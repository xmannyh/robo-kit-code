16 // BACKGROUND:
17 // When I first received an ultrasonic sensor I was not happy with how poorly
18 // it worked. Quickly I realized the problem wasn't the sensor, it was the
19 // available ping and ultrasonic libraries causing the problem. The NewPing
20 // library totally fixes these problems, adds many new features, and breaths
21 // new life into these very affordable distance sensors.
22 //
23 // FEATURES:
24 // * Works with many different ultrasonic sensors: SR04, SRF05, SRF06, DYP-ME007, URM37 & Parallax PING)))™.
25 // * Compatible with the entire Arduino line-up, Teensy family (including $19 96Mhz 32 bit Teensy 3.0) and derivatives.
26 // * Interface with all but the SRF06 sensor using only one Arduino pin.
27 // * Doesn't lag for a full second if no ping/echo is received.
28 // * Ping sensors consistently and reliably at up to 30 times per second.
29 // * Timer interrupt method for event-driven sketches.
30 // * Built-in digital filter method ping_median() for easy error correction.
31 // * Uses port registers for a faster pin interface and smaller code size.
32 // * Allows you to set a maximum distance where pings beyond that distance are read as no ping "clear".
33 // * Ease of using multiple sensors (example sketch with 15 sensors).
34 // * More accurate distance calculation (cm, inches & uS).
35 // * Doesn't use pulseIn, which is slow and gives incorrect results with some ultrasonic sensor models.
36 // * Actively developed with features being added and bugs/issues addressed.
37 //
38 // CONSTRUCTOR:
39 // NewPing sonar(trigger_pin, echo_pin [, max_cm_distance])
40 // trigger_pin & echo_pin - Arduino pins connected to sensor trigger and echo.
41 // NOTE: To use the same Arduino pin for trigger and echo, specify the same pin for both values.
42 // max_cm_distance - [Optional] Maximum distance you wish to sense. Default=500cm.
43 //
44 // SYNTAX:
45 // sonar.ping() - Send a ping and get the echo time (in microseconds) as a result.
46 // sonar.ping_in() - Send a ping and get the distance in whole inches.
47 // sonar.ping_cm() - Send a ping and get the distance in whole centimeters.
48 // sonar.ping_median(iterations) - Do multiple pings (default=5), discard out of range pings and return median in microseconds.
49 // sonar.convert_in(echoTime) - Convert echoTime from microseconds to inches (rounds to nearest inch).
50 // sonar.convert_cm(echoTime) - Convert echoTime from microseconds to centimeters (rounds to nearest cm).
51 // sonar.ping_timer(function) - Send a ping and call function to test if ping is complete.
52 // sonar.check_timer() - Check if ping has returned within the set distance limit.
53 // NewPing::timer_us(frequency, function) - Call function every frequency microseconds.
54 // NewPing::timer_ms(frequency, function) - Call function every frequency milliseconds.
55 // NewPing::timer_stop() - Stop the timer.
56 //
57 // HISTORY:
58 // 09/29/2015 v1.7 - Removed support for the Arduino Due and Zero because
59 // they're both 3.3 volt boards and are not 5 volt tolerant while the HC-SR04
60 // is a 5 volt sensor. Also, the Due and Zero don't support pin manipulation
61 // compatibility via port registers which can be done (see the Teensy 3.2).
62 //
63 // 06/17/2014 v1.6 - Corrected delay between pings when using ping_median()
64 // method. Added support for the URM37 sensor (must change URM37_ENABLED from
65 // false to true). Added support for Arduino microcontrollers like the $20
66 // 32 bit ARM Cortex-M4 based Teensy 3.2. Added automatic support for the
67 // Atmel ATtiny family of microcontrollers. Added timer support for the
68 // ATmega8 microcontroller. Rounding disabled by default, reduces compiled
69 // code size (can be turned on with ROUNDING_ENABLED switch). Added
70 // TIMER_ENABLED switch to get around compile-time "__vector_7" errors when
71 // using the Tone library, or you can use the toneAC, NewTone or
72 // TimerFreeTone libraries: https://bitbucket.org/teckel12/arduino-toneac/
73 // Other speed and compiled size optimizations.
74 //
75 // 08/15/2012 v1.5 - Added ping_median() method which does a user specified
76 // number of pings (default=5) and returns the median ping in microseconds
77 // (out of range pings ignored). This is a very effective digital filter.
78 // Optimized for smaller compiled size (even smaller than sketches that
79 // don't use a library).
80 //
81 // 07/14/2012 v1.4 - Added support for the Parallax PING)))™ sensor. Interface
82 // with all but the SRF06 sensor using only one Arduino pin. You can also
83 // interface with the SRF06 using one pin if you install a 0.1uf capacitor
84 // on the trigger and echo pins of the sensor then tie the trigger pin to
85 // the Arduino pin (doesn't work with Teensy). To use the same Arduino pin
86 // for trigger and echo, specify the same pin for both values. Various bug
87 // fixes.
88 //
89 // 06/08/2012 v1.3 - Big feature addition, event-driven ping! Uses Timer2
90 // interrupt, so be mindful of PWM or timing conflicts messing with Timer2
91 // may cause (namely PWM on pins 3 & 11 on Arduino, PWM on pins 9 and 10 on
92 // Mega, and Tone library). Simple to use timer interrupt functions you can
93 // use in your sketches totally unrelated to ultrasonic sensors (don't use if
94 // you're also using NewPing's ping_timer because both use Timer2 interrupts).
95 // Loop counting ping method deleted in favor of timing ping method after
96 // inconsistent results kept surfacing with the loop timing ping method.
97 // Conversion to cm and inches now rounds to the nearest cm or inch. Code
98 // optimized to save program space and fixed a couple minor bugs here and
99 // there. Many new comments added as well as line spacing to group code
100 // sections for better source readability.
101 //
102 // 05/25/2012 v1.2 - Lots of code clean-up thanks to Arduino Forum members.
103 // Rebuilt the ping timing code from scratch, ditched the pulseIn code as it
104 // doesn't give correct results (at least with ping sensors). The NewPing
105 // library is now VERY accurate and the code was simplified as a bonus.
106 // Smaller and faster code as well. Fixed some issues with very close ping
107 // results when converting to inches. All functions now return 0 only when
108 // there's no ping echo (out of range) and a positive value for a successful
109 // ping. This can effectively be used to detect if something is out of range
110 // or in-range and at what distance. Now compatible with Arduino 0023.
111 //
112 // 05/16/2012 v1.1 - Changed all I/O functions to use low-level port registers
113 // for ultra-fast and lean code (saves from 174 to 394 bytes). Tested on both
114 // the Arduino Uno and Teensy 2.0 but should work on all Arduino-based
115 // platforms because it calls standard functions to retrieve port registers
116 // and bit masks. Also made a couple minor fixes to defines.
117 //
118 // 05/15/2012 v1.0 - Initial release.
